# Engineering notes — append-only logs, NOT authoritative documentation

These are dated, point-in-time engineering logs (ship-*, path3-*, task-*, bench-*, …) kept for
historical record. They reflect what was true on their date and **may be stale**. Do not treat them
as a source of truth, and do not ground answers in them without re-verifying against the current
code.

**Authoritative documentation lives elsewhere:**

- [`ARCHITECTURE.md`](../../ARCHITECTURE.md) — the canonical map.
- [`docs/architecture/`](../architecture/) — code-traced subsystem deep dives.
- [`docs/adr/`](../adr/) — the durable decisions.
- [`howto/`](../../howto/) — the blueprint-author + operator guides.
- [`CONTRIBUTING.md`](../../CONTRIBUTING.md) — build / test / release / invariants.
- the code itself — when a note and the code disagree, **the code wins** (see the documentation
  plan's rules of engagement).

The five `*-design.md` / `*-open-questions.md` files kept under `docs/` (not here) are genuine
design documents, retained as candidates to fold into `docs/architecture/` or `docs/adr/`.
